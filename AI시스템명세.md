# DevReady AI 시스템 명세

> 면접 답변을 항목별로 평가하고 맞춤 질문·꼬리질문·종합 리포트를 생성하는 **대화형 모의면접 엔진**.
> 외부 API 호출이 아니라 **직접 파인튜닝·서빙한 모델 + RAG**로 IT 직무 특화 평가를 수행한다.

---

## 0. 팀원별 AI 담당

> DevReady의 모든 AI 기능은 **EXAONE Deep 7.8B 단일 모델**로 처리한다(외부 API 미사용). 음성 인식(Whisper)·표정 분석(face-api.js)은 EXAONE 채점 파이프라인에 입력을 공급하는 **보조 도구**다.

| 팀원 | AI 담당 기능 | 사용 기술 | 비고 |

### 모의면접 영상/음성 모드 분기

면접 환경에 따라 입력 방식이 갈린다.

| 환경 | 입력 | 담당 | 처리 |
|---|---|---|---|
| 영상 가능 | 카메라+마이크 | **박서진** | 시선·표정(face-api.js) + 음성 |
| 영상 불가 | 마이크만 | **박서진** | 음성 STT(Whisper) |

- 어느 환경이든 최종적으로 답변 텍스트는 **EXAONE 채점 파이프라인(정성채)** 으로 들어가 동일하게 평가된다.
- STT·표정·WPM·공백은 EXAONE에 **입력·부가지표를 공급**하는 단계이며, 점수 판정의 본체는 EXAONE이다.

## 1. 모델 — EXAONE Deep 7.8B

| 항목 | 내용 |
|---|---|
| 기반 모델 | `LGAI-EXAONE/EXAONE-Deep-7.8B` (revision `e3f42b18f6b1` 고정) |
| 유형 | 추론(reasoning) 모델 |
| 양자화 | 4-bit NF4 (VRAM 약 5GB) |
| 라이선스 | NC(비상업) — 연구·포트폴리오 용도 |

**선택 이유** — 추론 모델은 답을 내기 전 내부적으로 논리를 전개한다. 면접 답변의 근거·논리·일관성을 단계적으로 따져 *"왜 이 점수인지"* 까지 끌어낼 수 있어, 정답을 '생성'만 하는 일반 모델보다 평가에 적합하다. 한국어에 강하고 32B 대비 가벼워 RTX 4090 한 장으로 구동되며 비용·시간을 절감한다.

---

## 2. 시스템 아키텍처

```
입력                  FastAPI 서빙            EXAONE Deep 7.8B           구조화 출력
질문·답변       →      요청 검증·라우팅   →    4-bit + QLoRA 어댑터   →   JSON (평가·질문·리포트)
이력서·공고                                          ↕
                                              RAG · 지식 그라운딩
                                           bge-m3 임베딩 + FAISS
                                           (질문 은행 한 5,684 / 영 134)

음성 입력(STT)  →  Whisper large-v3  →  텍스트  →  동일 채점 파이프라인
```

- 서빙: **FastAPI** (port 8000)
- 환경: **RunPod RTX 4090** · 100GB 네트워크 볼륨 · 모델 약 5GB(4-bit)

---

## 3. RAG (검색 기반 지식 보강)

- **임베딩** bge-m3 + **FAISS** 인덱스, 질문 은행 한국어 5,684개
- **의미 기반 검색** — 단어가 달라도, 음성 전사 특유의 음차 표기도 *뜻으로* 찾아 환각을 줄인다
- **유사도 필터링** — 관련도 점수 미달 자료는 프롬프트에서 제외해 주제 이탈을 차단 (예: '스택·큐' 질문에 '그래프'가 끼어들던 문제 해결)
- 모델 재학습 없이 지식 추가·교체 가능

---

## 4. 파인튜닝 — QLoRA

| 구성 | 값 |
|---|---|
| LoRA rank / alpha | r=16 / α=32 |
| 학습 | 3 epoch |
| loss | 2.92 → 0.83 |
| 어댑터 크기 | 168 MB |

작은 '어댑터'만 학습해 단일 4090으로 7.8B를 미세조정한다. 원본은 보존하고 어댑터만 교체하므로 개선·롤백이 빠르다.

**핵심 — 목표는 '점수 정확도'가 아니라 '형식·행동 안정화'다.** base 모델은 평가를 시키면 깨진 JSON·누락된 필드·전부 0점 같은 출력을 자주 낸다. 어댑터는 **항상 같은 스키마로, 파싱 가능한, 일관된 채점**을 내도록 행동을 교정하는 것이 목적이다.

---

## 5. 자가증류 학습 데이터 + 3중 안전 가드레일

점수 라벨이 없으므로, **모델이 스스로 채점 데이터를 만들되 '검증'을 거쳐** 학습에 사용한다.

```
질문 + 답변  →  EXAONE 채점(루브릭+RAG)  →  점수 + 피드백 JSON  →  검증 통과분만 학습 데이터
```

**3중 가드레일**
1. **overall 점수 불신** — 모델이 낸 종합 점수를 믿지 않고 4개 하위 점수의 평균으로 재계산
2. **전부 0점 표본 제외** — JSON 틀만 복사하고 채점하지 않은 '깡통' 출력은 학습에서 배제
3. **3조건 통과만 채택** — `</thought>` 완료 + JSON 파싱 성공 + 한국어 추론 확인

> 학습 데이터: AI Hub '채용 면접 인터뷰' 데이터셋 기반. **원본은 재배포하지 않으며, 방법·수치만 문서화한다.**

---

## 6. 한국어 추론 안정화

EXAONE은 내부 추론이 영어로 흐를 수 있다. **prefill 기법**으로 사고를 한국어로 강제한다 — 채팅 템플릿 끝의 `<thought>` 뒤에 한국어 문장을 덧붙여 토크나이즈하면, 이후 추론이 한국어로 이어진다. (한국어 여부 판정은 prefill을 제외한 생성분으로 한다.)

## 7. 멀티모달 (보조 — 핵심은 자체 엔진)

| 기능 | 도구 | 설명 / 한계 |
|---|---|---|
| 음성 인식 (STT) | faster-whisper large-v3 | 답변을 텍스트로 받아쓰기. 일반 Whisper보다 빠르고 가벼움 / 소음·발음에 약해 오인식 가능 |
| 음성 합성 (TTS) | TTS | AI 질문을 사람 목소리로 읽어 실제 면접처럼 진행 / 합성 음성이 어색할 수 있음 |
| 표정·시선 분석 | face-api.js | 카메라로 표정·시선 분석. **브라우저에서 동작해 영상 외부 유출 없음(점수만 백엔드 전송)** / 조명·각도에 민감, 감정 추정은 참고용 |

## 8. 이중언어 설계 (한 / 영)

요청의 `lang`으로 분기, 기본값 `ko` → 기존 한국어 동작 완전 보존.

| 기능 | 한국어 (ko) | 영어 (en) |
|---|---|---|
| 질문 검색 (RAG) | 5,684개 | 134개 (큐레이션) |
| 답변 평가 | QLoRA 어댑터 | base 모델 + 영어 루브릭 |
| 추론 언어 유지 | 한국어 prefill 강제 | 네이티브 (트릭 불필요) |
| 질문생성·꼬리질문·리포트·STT | 지원 | 지원 |

**핵심 통찰** — 영어는 EXAONE이 추론을 네이티브로 하므로 별도 어댑터가 불필요하다. base 모델 + 0~100 스케일을 명시한 영어 루브릭만으로 한국어 수준의 평가를 달성한다.

## 11. 운영·안정화

- **GEN_LOCK** — 동시 요청을 직렬화해 GPU(CUDA) 충돌 방지
- **Budget Forcing** — 추론·답변 분량 상한 + 실시간 스트리밍으로 1~2초 내 첫 응답, Cloudflare 524 타임아웃 회피
- **재현성** — 모델·핵심 라이브러리를 특정 버전(revision)으로 고정해 업데이트로 결과가 바뀌는 것을 차단
- **입력 검증·클램핑** — 비정상·과도한 길이 입력을 사전 검증하고 점수를 정해진 범위로 강제 → 깨진 출력 없이 일관된 응답

---

## 12. 한계 및 고려점

- **상용 불가** — NC 라이선스로 실제 서비스엔 사용 불가 (연구·포트폴리오 한정)
- **추론 지연** — 추론 모델 특성상 응답이 느려질 수 있어 Budget Forcing·스트리밍으로 보완
- **평가의 한계** — 정성 평가는 정답이 하나가 아니므로 **참고 지표로 활용**, 사람 판단을 대체하지 않음
- **데이터 의존성** — QLoRA는 전체 미세조정보다 미세 성능이 낮을 수 있고 학습 데이터 품질에 의존 (자가증류 검증으로 완화)

## 기술적 의의 · 차별점

- 외부 API 호출이 아닌 **자체 호스팅 + 파인튜닝** — 모델·데이터를 직접 통제
- **QLoRA로 모델 행동을 직접 교정** (깨진 JSON → 일관된 스키마)
- **RAG + 자가증류 데이터 파이프라인** 직접 설계
- **4-bit 양자화로 단일 GPU 서빙** — 저비용
- **정량 비교로 개선 효과 입증** (loss 2.92 → 0.83)
- **데이터 라이선스·프라이버시 고려** (AI Hub 원본 비배포, 표정 분석 브라우저 처리)
